# 地圖縮放渲染修正計畫

## 問題描述
在進行地圖縮放後，畫面上左上角區域的物件（實體）會消失不見，但主城（紅圈處）附近的物件正常顯示。這通常是座標轉換（Coordinate Transformation）或剔除邏輯（Culling Logic）在縮放倍率改變時計算錯誤所致。

## 執行步驟
1. **分析現有渲染架構**：
   - 檢查 `src/systems/RenderSystem.js` 或負責地圖渲染的相關檔案。
   - 找出處理「可視區域剔除」(Culling) 的邏輯。
   - 檢查縮放 (Zoom/Scale) 如何影響這些計算。
2. **重現與定位問題**：
   - 查看地圖縮放時，剔除框 (Culling Bounds) 的計算公式。
   - 檢查是否有偏移量 (Offset) 未能正確隨縮放比例調整。
3. **實施修正**：
   - 修正剔除框的計算公式，確保其在不同縮放層級下皆能準確涵蓋可見區域。
   - 確保所有座標轉換（世界座標 vs 螢幕座標）皆考慮了縮放倍率。
4. **驗證與測試**：
   - 運行遊戲並進行縮放測試，觀察左上角區域物件是否出現。
   - 確保效能符合規範（Draw Calls < 50, FPS 60）。
   - 執行 `npm run finalize` 完成任務。

## 預期結果
縮放後地圖各個區域的物件均能正常顯示，且無破圖或渲染錯誤。
