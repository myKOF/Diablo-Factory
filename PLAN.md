# 物流線路徑優化計畫

## 目標
優化物流線（輸送帶）在拖曳時的 L 形路徑生成邏輯，使其能夠根據玩家游標移動的先後順序（方向）來決定彎折路徑，而非固定優先順序。

## 實施步驟
1. **分析現有邏輯**：
   - 閱讀 `src/systems/ConveyorRouter.js` 中的 `getLShapePath` 函式。
   - 了解目前是如何決定先走 X 軸還是 Y 軸。

2. **追蹤游標移動順序**：
   - 在 `src/systems/ConveyorSystem.js` 中，記錄拖曳開始後的第一個有效移動方向。
   - 或是根據目前的游標位置與起點的偏移量，判斷玩家「更傾向」於哪種路徑。
   - 根據使用者需求，關鍵在於「先向上拉再向左拉」與「先向左拉再向上拉」的區分。

3. **修改路徑生成參數**：
   - 讓 `ConveyorRouter.getLShapePath` 接受一個額外的參數（例如 `preferredFirstAxis` 或 `dragHistory`），用以決定優先路徑。

4. **整合與驗證**：
   - 更新 `ConveyorSystem` 的 `updateDragNow` 邏輯，將方向意圖傳遞給 Router。
   - 確保所有端口（上下左右）都能正確套用此邏輯。

5. **自動測試與驗證**：
   - 執行 `npm run finalize` 確保系統穩定。

## 預期結果
- 當游標從端口先垂直移動再水平移動時，物流線呈現「直->橫」。
- 當游標從端口先水平移動再垂直移動時，物流線呈現「橫->直」。
