## [UI 動態配置與架構解耦協議]
- **總司令模式 (Commander Pattern)**：`src/ui/ui.js` 僅作總司令，專職初始化介面、攔截全域點擊事件與路由分發。嚴禁在此撰寫面板細節邏輯。
- **單一職責拆分 (Component Decoupling)**：所有子介面必須獨立為單一腳本，專職處理自身渲染與狀態，由總司令負責呼叫。
- **動態參數化 (Config-Driven)**：禁止數值寫死。所有 UI 新參數必須寫入 `ui_config.js` 並附帶詳細繁體中文註釋。
- **SSOT 唯一來源**：UI 屬性必須讀取自 `ui_config.js`。新增組件優先繼承既有風格。
- **風格自動對齊**：新增界面前需透過安全搜尋檢索現有代碼，確保邊角半徑、陰影與間距 100% 吻合。

## [架構解耦與防污染搬移協議 (Architecture & Anti-Pollution Protocol)]
- **職責絕對分離 (Strict Separation)**：嚴禁在邏輯層 (Systems) 中直接操作渲染物件 (Phaser Sprites/UI)。
- **事件驅動通訊 (Event-Driven)**：跨系統互動必須透過全域事件總線 (EventBus/EventEmitter) 解耦，嚴防循環依賴。
- **無損搬移三步曲 (Safe Migration)**：並行建置 -> 路由切換 -> 安全銷毀。**嚴禁「先刪除舊代碼再重寫」**。
- **依賴爆炸檢查 (Dependency Grep)**：修改核心 API 或移動檔案前，強制全域檢索同步更新依賴。
- **狀態防篡改 (State Encapsulation)**：全域遊戲狀態必須嚴格封裝，統一透過 Action 或 Setter 變更。

## [物流生產線系統開發協議 (Logistics & Conveyor System)]
**【1. 核心定義】**
- **物流線**：具方向性的產品傳輸線。
- **端口**：建築輸出/輸入產品的出口或入口。
- **產品**：建築生產並藉由物流線運輸的物品。
- **物流線延伸**：由既有物流線向兩側或原方向拉出新線段，完成後自動合併。
- **接通與中斷**：由建築端口拉出至另一端口且無斷點為「接通」；缺少起終點端口或有斷點為「中斷」。兩狀態可互轉。
- **匯合點與合併**：多條物流線匯聚為一的位置為「匯合點」；多條線拼接為新線的過程為「合併」。
- **主線與支線**：多線匯合後即為主線（含其直線方向線段），匯合點兩側接入的為支線。
- **群組**：連續線段為一群組。有分支時，主線所有線段同群組，兩側分支線段各自為獨立群組。
- **建築虛影**：建造前的軌跡標示狀態。

**【2. 基礎操作】**
- **端口建造**：於建築端口左鍵拉出虛影，放開後沿軌跡建立順向物流線。
- **延伸建造**：於既有物流線上任意點左鍵拉出虛影，放開後建立順向物流線。
- **場景建造**：激活建造按鈕後，於無障礙空白處左鍵拉出虛影建立獨立線段（此模式亦兼容上述兩者）。
- **全選**：對物流線左鍵雙擊，全選該「群組」所有線段。
- **刪除與復原**：Delete 鍵刪除所選線段；Ctrl+Z 復原上一步，最多回復 5 步。
- **合併操作**：A線起點拉至C線終點建立B線，ABC將合併為一條線，群組歸屬依主/支線規則判定。

**【3. 建造與運行規則】**
- **防分叉與單一方向**：物流線可合併不可分叉。強行分叉時，以「最新拉出的線段」為最終線段方向。每段僅限單一方向。
- **網路重算**：多線匯合時，觸發整個相連物流線網路的重算與重繪。
- **防穿透與斷點**：不可穿過建築，最多延伸至端口。穿過端口後方不顯示虛影。拉伸時若跨越其他物流線/障礙點，視為「多條單獨線段」，不與被跨越者合併。
- **視覺指示**：每線段必須有方向指示箭頭。
- **狀態顏色**：中斷=灰色；接通=亮青色；運輸中=藍色。不允許除匯合點以外的重疊。
- **物品回流與銷毀**：運輸中物流線被刪除（含Ctrl+Z）時，產品退回原建築。若空間/堆疊已滿，直接銷毀產品。
- **匯合點輪詢機制 (Round-Robin)**：匯合點採「先到先過」。若主線堵塞，產品於匯合點前一格等待。此時匯合點將「輪流放行」各線產品，無任何線路有絕對優先權。
- **平滑運動演算**：直線為等速運動，支線向主線轉彎時必須是「平滑弧線等速運動」，嚴禁視覺上的加減速頓挫感。
- **防呆與合併限制**：若合併將導致方向衝突（方向相反），系統禁止該次合併。

**【4. 底層效能強制規範 (Performance Critical)】**
- **陣列偏移運輸法**：嚴禁為每個產品掛載獨立 Update()。相連物流線合併為「運輸陣列」，物品僅紀錄 Index 與 Offset。
- 每一幀僅更新 Offset，達成萬級物品無損運輸。

## [代碼收斂與自動化驗證]
- **代碼洗滌**：任務結束前，刪除未引用變數與除錯 Log。
- **唯一指定測試框架 (Playwright Only)**：所有自動化測試、E2E 驗證與 UI 互動檢測，**強制且唯一指定使用 Playwright**。嚴禁 Agent 擅自安裝或使用 Cypress, Selenium, Puppeteer 等工具。
- **按需自測**：預設不進行自測（參見動態模式協議）。若觸發自測，腳本必須強制執行 `await browser.close();` 關閉網頁實體，並以 `finally { process.exit(); }` 徹底了結進程。

## [動態執行模式切換協議 (Conditional Protocol)]
- **預設模式 (Default)**：常規靜默開發，不觸發自動化測試與即時監控。
- **一次性自測 (觸發指令：`[自測]`)**：任務完工後，強制啟動 Playwright 進行驗證並回報。完成後自動回歸「預設模式」。
- **一次性監控 (觸發指令：`[進入監控模式]`)**：暫停代碼編寫，監聽 `GameEngine.state` 並實時輸出 `logisticsLines` 的狀態比對與完整性報告至控制台。任務完成後自動回歸「預設模式」。

## [任務完成強制收尾協議 (MANDATORY FINALIZATION PROTOCOL)]
- 任務完成的唯一標準：成功執行 `npm run finalize`。Agent 不得自行判定結束。
- 所有任務（包含自測）完成後，必須執行 `npm run finalize`。未執行即視為失敗。

## [無痕測試與自動清理協議]
1. **領地限制 (Sandbox)**：測試產出物統一存放於 `/tmp/`，嚴禁置於 `src/` 或根目錄。
2. **Git 守門員**：執行 `git add` 前，必須清理 `.gitignore` 以外的測試遺留檔。
3. **自動銷毀機制**：變數宣告必須在最外層，`finally` 區塊強制執行 `browser.close()` 與 `fs.rmSync('tmp/', ...)`。
4. **無感日誌**：除錯用的 `console.log` 通過驗證後自動抹除。
